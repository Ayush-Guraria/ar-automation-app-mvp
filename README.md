# AR Automation

A self-service Streamlit app for accounts-receivable teams: upload an invoice
CSV, get every invoice automatically classified into a risk tier by days
overdue, and draft a policy-grounded follow-up email for any invoice with one
click. Email drafts are generated by Claude, grounded via retrieval-augmented
generation (RAG) over your internal AR payment-terms policy, so every draft
cites the specific policy language that justifies its tone and urgency.

## Features

- CSV upload with an invoice preview table
- Risk-tier classification with color coding (green/yellow/orange/red for
  Current / At Risk / Overdue / Critical)
- Dashboard with four metric cards: total outstanding, invoice counts by
  tier, average days overdue, total critical amount
- "Draft Follow-Up Email" button per invoice, calling Claude with RAG-retrieved
  policy context
- Structured Claude output: subject line, email body, tone, cited policy,
  urgency level
- SQLite audit trail logging every email-draft action
- CSV export of the classified invoice table

## Run locally

```
python generate_ar_data.py
streamlit run app.py
```

The first command writes `sample_ar_invoices.csv` (20 synthetic invoices) and
seeds the local ChromaDB policy index. The second starts the app — upload the
generated CSV (or your own, matching the required columns below) to get
started.

## Required columns for your own CSV

`invoice_number`, `client_name`, `amount_due`, `due_date`, `days_overdue`
(`client_email` and `invoice_date` are optional but included in the generated
sample data).

## Required secrets

Copy `.streamlit/secrets.toml.template` to `.streamlit/secrets.toml` and set:

```toml
ANTHROPIC_API_KEY = "sk-ant-..."
CLAUDE_MODEL = "claude-sonnet-4-5"   # optional, this is the default
```

For local development you can instead put `ANTHROPIC_API_KEY` in a `.env`
file — it's picked up automatically if `st.secrets` has no key set.

On Streamlit Community Cloud, set the same keys in the app's **Settings →
Secrets** panel instead of committing `secrets.toml`.
